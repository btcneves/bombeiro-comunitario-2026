# FAQ

## Preciso saber programar?

Não. O projeto usa Markdown e Obsidian, não desenvolvimento de software
tradicional para o uso diário.

## Preciso usar Obsidian?

Para aproveitar o sistema completo, sim. Parte do valor está nas consultas,
links internos e plugins do ecossistema Obsidian.

## Posso usar no celular?

Sim. O vault pode ser aberto no aplicativo móvel do Obsidian, desde que os arquivos estejam acessíveis no dispositivo.

## Como adapto para outro concurso?

Substitua edital, reorganize disciplinas, recrie notas atômicas e ajuste
dashboard, cronograma e README. O guia completo está em
`docs/adaptando-para-outros-concursos.md`.

## Como sei o que revisar hoje?

Pelo `🎯 Dashboard` e pelos índices de revisão. O campo `próxima-revisão` controla o que entra na fila do dia.

## Como usar flashcards?

Abra os decks ou notas que contêm cards e revise com o plugin Spaced
Repetition. O formato principal é `pergunta::resposta`.

## O que fazer quando erro uma questão?

Registrar o erro, identificar a causa, corrigir a nota relacionada e reforçar com revisão e flashcards.

## O que significa nível-domínio?

É a maturidade do tópico no sistema: `0` não estudado, `1` visto, `2` revisado, `3` dominado.

## Posso compartilhar este projeto?

Sim, respeitando a licença MIT e deixando claro que o material é apoio de estudo, não substituição de fontes oficiais.

## Este material substitui o edital?

Não. Em caso de divergência, o edital e as fontes normativas oficiais prevalecem sempre.

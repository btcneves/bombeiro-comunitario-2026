# Security Policy

## Escopo de segurança

Este projeto é, hoje, um repositório documental e um vault Obsidian. Ele não processa dados sensíveis, não oferece serviço online, não expõe API e não depende de infraestrutura de produção.

## O que reportar

Mesmo assim, vulnerabilidades ou riscos futuros podem surgir em:

- scripts adicionados posteriormente
- automações de build ou publicação
- integrações com GitHub Pages
- exportadores de flashcards ou ferramentas auxiliares

Se identificar um problema de segurança plausível, abra uma issue descrevendo:

- o componente afetado
- o cenário de risco
- como reproduzir
- o impacto estimado

## Tratamento esperado

Relatos serão avaliados de acordo com a severidade e com o escopo real do projeto no momento do reporte.

# Contribuindo

Obrigado por contribuir com o projeto `bombeiro-comunitario-2026`. Este repositório é ao mesmo tempo um sistema real de estudo em Obsidian e um case técnico de organização de conhecimento. Toda contribuição deve preservar esses dois objetivos.

## Princípios de contribuição

- Não recomece o vault e não desestruture a organização existente.
- Não sobrescreva notas de estudo sem justificativa clara.
- Prefira melhorias pequenas, específicas e rastreáveis.
- Mantenha consistência de nomes, links internos e padrão Markdown.
- Documente mudanças que alterem fluxo, metodologia ou navegação.

## Como sugerir melhorias

Abra uma issue quando quiser:

- propor melhora na documentação
- relatar inconsistência entre arquivos
- sugerir melhoria na arquitetura do vault
- ajustar metodologia de revisão ou flashcards
- propor material visual para GitHub Pages ou portfólio

Use os templates disponíveis em `.github/ISSUE_TEMPLATE/`.

## Como adicionar novas notas

Ao criar conteúdo novo no `Obsidian Vault`:

- mantenha a disciplina correta em `02 - Disciplinas`
- crie ou atualize a nota atômica em `03 - Tópicos Atômicos`
- preserve o frontmatter padrão
- relacione o conteúdo a revisões, flashcards e erros quando fizer sentido
- evite notas gigantes; prefira recortes conceituais menores

## Padrão Markdown

Diretrizes mínimas:

- use títulos claros e consistentes
- mantenha listas objetivas
- prefira links internos do Obsidian para relacionar conceitos
- preserve acentuação e nomes originais das pastas
- evite duplicar informação que já exista em nota mais apropriada

## Como adaptar para outro concurso

Contribuições voltadas à reutilização do projeto devem:

- separar o que é metodologia do que é conteúdo específico do concurso
- preservar a estrutura-base do sistema
- atualizar documentação pública para refletir a nova finalidade
- incluir instruções de migração no pull request

Consulte também [docs/adaptando-para-outros-concursos.md](docs/adaptando-para-outros-concursos.md).

## Pull requests

Antes de abrir um pull request:

- revise links e nomes de arquivos
- confirme que a estrutura do vault continua íntegra
- atualize documentação relacionada
- descreva claramente o impacto prático da mudança

Checklist mínimo:

- documentação clara
- links funcionando
- sem quebrar estrutura do vault
- conteúdo revisado

## Conduta esperada

Ao participar, siga o [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Discussões devem ser técnicas, respeitosas e orientadas à melhoria do projeto.
